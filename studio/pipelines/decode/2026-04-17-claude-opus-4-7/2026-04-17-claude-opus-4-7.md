# Claude Opus 4.7 不是顶配——顶配被 Anthropic 锁起来了

**4 月 16 日，Anthropic 发布 Claude Opus 4.7。这个版本的 SWE-bench Verified 分数是 87.6%，超过了所有公开可用的模型。但它不是 Anthropic 手上最强的——最强的那个叫 Claude Mythos Preview，SWE-bench Verified 考了 93.9%，然后被锁进了一个 11 家科技巨头组成的"玻璃翼联盟"（Project Glasswing），普通开发者拿不到。**

这是第一次，有厂商主动把"最强模型"藏起来，只给"第二强"发广泛许可证。这件事比 Opus 4.7 本身更值得拆开看。

![封面](00_cover.png)

## 一、开场：Anthropic 发了一个"刻意降配"的版本

先看原文怎么说的。Anthropic 在公告里有一段措辞非常罕见：

> "Opus 4.7 is the first such model: its cyber capabilities are not as advanced as those of Mythos Preview (indeed, during its training we experimented with efforts to differentially reduce these capabilities)."
>
> （Opus 4.7 是第一个这样的模型：它的网络安全能力不像 Mythos Preview 那么先进——事实上，在训练过程中，我们专门尝试了差异化降低这些能力。）

请注意"differentially reduce"（差异化降低）这个词。这不是"新模型恰好能力更弱"，而是"我们主动在训练阶段把某些能力切掉"。这是一场训练侧的外科手术。

配合另一个事实一起看：就在一周前，Anthropic 发布了 Mythos Preview，但没对公众开放，只给了 Project Glasswing 联盟——AWS、Apple、Google、Microsoft、NVIDIA、JPMorganChase、Cisco、CrowdStrike、Broadcom、Palo Alto Networks、Linux Foundation 这 11 家。理由是 Mythos 在训练中"无需工程师指导就自主发展出了下一代攻击性网络能力"，可以找到全球关键基础设施里隐藏的漏洞。

把两件事串起来，你会发现 Anthropic 正在开创一种新的模型发布范式：**最强的那个不公开，能公开的是被刻意阉割过的版本。** 这篇解读要讲的第一件事就是：Opus 4.7 不是"Anthropic 的新旗舰"——它是 Anthropic 愿意让你碰的最强模型。

## 二、三个转向：为什么"稍微强一点"就值得写篇公告

把 Mythos 的影子放一边，单看 Opus 4.7 相对 4.6 的提升，其实并不以"benchmark 峰值"为主线。真正值得从业者盯住的是三个非常具体的方向转变。

### 转向一：从"能力天花板"到"token 效率"

原文里埋了一句听起来轻描淡写但杀伤力巨大的话：Opus 4.7 在 `xhigh` 效果档位跑 100k token，性能已经等同于 Opus 4.6 在 `max` 档位跑 200k token。

换句话说，**同样的任务质量，token 消耗减半**。对任何一个在生产环境部署 Claude 的公司来说，这意味着直接的成本折半——定价没变（输入 $5/M，输出 $25/M），但你现在用一半的 token 就能达到同样的效果。

这里面有一个没说出口的假设：过去几代模型的竞争焦点是"跑得多远"（更长上下文、更高 benchmark 峰值），但 Opus 4.7 第一次把"跑得省"摆到了优先级前列。Replit 的 CEO Michele Catasta 在官方 testimonial 里说得很直白："same quality at lower cost"——同样的质量，更低的成本。

这个转向背后是行业阶段的切换。大模型的 price-performance 曲线已经从 2023-2024 年的"能力陡峭上升期"进入"效率边际优化期"。Anthropic 在博客里说"Opus 4.7 在 Rakuten-SWE-Bench 上解决了 3 倍于 Opus 4.6 的生产任务"——注意，他们不说"benchmark 分数高了多少"，而说"通过的真实任务多了几倍"。衡量单位从 percentage point 换成了 task throughput。

### 转向二：从"一次响应"到"数小时持续工作"

Cognition（Devin 背后的公司）的 CEO Scott Wu 对 Opus 4.7 的评价是："It works coherently for hours, pushes through hard problems rather than giving up"——它能连贯工作好几个小时，遇到硬问题不放弃。

"Hours"这个词在这里不是随便说的。过去一年里，agentic AI 的最大瓶颈从来不是"能力不够聪明"，而是"跑一会儿就偏航"。上下文累积超过 50 页，模型开始忘记初始目标；处理 3-4 层嵌套调用，决策链开始自我矛盾；无监督运行超过半小时，结果越跑越离谱。

Opus 4.7 给这个问题发了两个新武器：

**Task Budget**（任务预算）。这是一个公测功能——你给 Claude 一个大致的 token 预算（覆盖思考、工具调用、工具返回、最终输出），模型会看到一个"剩余预算"的 running countdown，像倒计时一样，并据此决定优先级。余额够就深入探索，余额紧就收敛收尾。

**Auto Mode**（自主模式）。只开放给 Claude Code 的 Max 用户，允许 Claude 在权限范围内自己做决策、不打断用户。配合 Task Budget，相当于给 agentic loop 装上了自动驾驶——定目标、给预算，让它自己跑几个小时。

这两个功能合起来说明一件事：**Anthropic 在把 agent 产品化的颗粒度从"一轮对话"推到"一个任务"。** 他们不再把 Claude 卖成"你问我答的助手"，而是卖成"你交任务我做几小时的同事"。

![三个转向](02_three_shifts.png)

### 转向三：从"模型即终点"到"命令化工作流"

Opus 4.7 带来的 `/ultrareview` 命令是这一转向最明显的信号。这不是模型升级——这是**把一个特定工作流打包成了命令**。

`/ultrareview` 会对代码改动做一次结构化审查，从架构、安全、性能、可维护性几个维度找单次 review 容易漏掉的深层问题。CodeRabbit 在官方 testimonial 里说 Opus 4.7 是"the sharpest model we've tested"，recall 提升超过 10%。

有意思的地方在于 Pro 和 Max 用户每月送 3 次免费使用——这是一个极具信号的商业动作。Anthropic 在试探："用户愿不愿意为单次 high-intensity 工作流单独付费？" 如果试探成功，未来很可能看到 `/refactor`、`/migrate`、`/threat-model` 等一堆基于相同模式的命令陆续上线。

这一转向的隐含判断是：**模型的差异化红利在变薄，产品层的差异化红利在变厚。** 同一套模型能力，谁把它组织成更好用的工作流，谁就赢。Cursor、Windsurf、Cognition 这些早就在做这件事；Anthropic 现在亲自下场，意味着它认为这个战场重要到不能假手他人。

## 三、Benchmark 真相：亮眼数字背后的层次

Opus 4.7 公布的 benchmark 数字确实好看：

| 指标 | Opus 4.7 | Opus 4.6 | 对比 |
|---|---|---|---|
| SWE-bench Pro | 64.3% | 53.4% | +10.9pt |
| SWE-bench Verified | 87.6% | 80.8% | +6.8pt |
| CursorBench | 70% | 58% | +12pt |
| Rakuten-SWE-Bench（生产任务） | 3x | 1x | 3 倍 |
| Visual-acuity benchmark | 98.5% | 54.5% | +44pt |
| Finance Agent（General Finance） | 0.813 | 0.767 | +0.046 |

看起来是全面碾压。但如果你把 Mythos Preview 的分数也放进来对比，故事就不一样了：

![Benchmark 分层](03_benchmarks.png)

> Mythos Preview 在 SWE-bench Verified 上是 **93.9%**，比 Opus 4.7 高 6.3 个百分点。在 Anthropic 官方公布的基准表里，Mythos 在 agentic coding、网络安全漏洞复现、工具使用等绝大多数项目上都是最高分——但这一栏只出现在"展示用的对比表"里，不出现在"你能买到的产品线"里。

这带来一个很有意思的变化：**benchmark 榜单不再是单纯的能力展示，它成了一个战略博弈工具**。 Anthropic 把 Mythos 的分数放进对比表里不是为了宣传，而是为了说明"我们手上的还有更强的，但我们选择不放出来"。这是一种 capability signaling + restraint signaling 的组合拳——既让对手知道实力，又让监管者看见克制。

原文里 The Next Web 的报道提到 Opus 4.7 在公开 benchmark 上同时超过了 OpenAI 的 GPT-5.4 和 Google 的 Gemini 3.1 Pro，差距不小。但 Anthropic 没有把这放在公告的显眼位置——他们更关心的不是跟对手比，而是强调"我们的 tier-2 模型已经足够强"这个叙事。

## 四、差异化训练：一个被轻描淡写的技术转折

回到原文那句"differentially reduce"。这是整个公告里技术密度最高的一个词，但 Anthropic 把它藏在段落中间，没展开讲。

差异化训练意味着什么？意味着一个模型可以在训练阶段被"手术式"地削弱某些能力，同时保留其他能力。这在过去是非常难做的——能力之间是耦合的，砍一刀通常会伤到相邻的肌肉。Anthropic 在这里宣称他们做到了精准切除：Mythos 的攻击性网络能力被专门降低，但编码、推理、视觉能力仍然保留。

这件事的副作用已经显现。公告的安全评估部分有一行相当诚实的承认：Opus 4.7 在"controlled substance harm-reduction advice"（管制物质减害建议）上**比 Opus 4.6 稍差**——更容易提供"过度详细的减害建议"。这说明差异化训练的精度还没到"只砍该砍的"，它会波及到意图相近但性质不同的能力类别。

这是一个被低估的 AI 安全技术节点。如果差异化训练成熟，未来可能看到的模式是：**一个底座模型，训练出 N 个版本，每个版本针对不同行业/场景削减不同的风险能力**。企业版砍掉娱乐类，医疗版砍掉金融类，教育版砍掉安全类。模型定制化的颗粒度从 prompt 层下沉到训练层。

Anthropic 没明说，但 Project Glasswing 的存在就是这个范式的一次预演——对 AWS、Google、Microsoft 这些巨头，给的是"未手术版"（Mythos）；对普通开发者，给的是"减害版"（Opus 4.7）。同一个底座，两种产品。

## 五、一个新概念：Model Tiering

把 Mythos Preview + Project Glasswing + Opus 4.7 + Cyber Verification Program 这几件事串起来看，Anthropic 实际上开创了一个我称之为 **Model Tiering（模型分级发布）** 的新范式。

过去 LLM 行业的默认假设是"最新就是最强，最强就是面向公众的"。GPT-4 Turbo 比 GPT-4 强，所以替换 GPT-4；Gemini 1.5 Pro 比 1.0 强，所以替换 1.0。发布即公开，公开即最强。

Model Tiering 打破了这个假设。它把模型分成三个层级：

**Tier 0 — 内部不发布**：能力过强但风险不可控的模型（目前没有公开案例，但 Anthropic 的 RSP 框架里有对应描述）。

**Tier 1 — 联盟限制发布**：Mythos Preview 这一级。能力最强但只给 vetted consortium。用户知道它存在、知道它更强，但大多数人拿不到。

**Tier 2 — 广泛商业发布**：Opus 4.7 这一级。经过差异化降能处理、加了自动安全护栏、可以公开卖 API。

**Tier 3 — 专业豁免**：Cyber Verification Program 里的安全从业者，他们用 Opus 4.7 但解锁一部分被默认关闭的能力（比如漏洞研究、渗透测试、red-teaming）。

这个四层结构让 Anthropic 一次性解决了几个矛盾：**既能向投资人展示前沿能力，又能向监管者展示克制；既能保留商业化的广泛可及性，又能满足安全关键场景的专业需求。** 这是一个精巧的产品分层。

OpenAI 和 Google 目前都还没有类似的公开结构。这可能意味着两种解读：要么它们没这么强的模型，所以不需要这种分层；要么它们有但还没找到对外陈述这件事的框架。无论哪种，Anthropic 的这次发布都在行业里立了一个 precedent——未来如果 OpenAI 发了一个"GPT-X 有限释放版"，我们知道它的剧本是谁写的。

## 六、盲区：这次发布没说的事

有几件事值得专门拎出来看 Anthropic **没说**的部分。

**第一，Mythos 的能力上限没公开。** 官方只给了 SWE-bench Verified 的 93.9% 和几个基准数字。Mythos 在"autonomously developed next-gen offensive cyber capability"这件事上具体能做到什么程度？能自动发现 0day 吗？能生成可用的 exploit 吗？官方只做了定性描述，没给定量边界。这是 Anthropic 有意保留的信息暗场。

**第二，"数小时连贯工作"没有时长上限。** Cognition 说 hours，但具体多少小时？3 小时还是 8 小时？模型在 4 小时后的任务完成度相比 1 小时后下降多少？这些运营级指标都没公布。这让"hours"听起来是一个营销词而不是工程指标。

**第三，价格没降。** 输入 $5/M，输出 $25/M，和 Opus 4.6 一样。但如果 xhigh 100k 的性能等同 max 200k，用户实际付费会减半。这是一种"隐性降价"——定价不动，性价比靠 token 效率提升消化。这让 Anthropic 在面对"Claude 为什么比 GPT 和 Gemini 贵"的质疑时，有了"同样任务我们用更少 token"的话术挡箭牌。

**第四，差异化训练的技术细节没公开。** "我们尝试了差异化降能"——但用什么方法？RLHF 变种？constitutional AI 的扩展？specific capability ablation？这个可能是 Anthropic 接下来两年最核心的技术护城河，他们不会轻易给出。

## 七、对 AI 从业者意味着什么

把以上都整理完，对不同角色的从业者有具体的行动提示：

**对 AI 产品经理**。重新设计你的 agentic 工作流的"任务单位"。过去你可能把一个复杂任务拆成 20 轮对话，每轮 5 分钟；现在可以考虑交给 Claude 一次 2 小时的任务，用 Task Budget 来控制上限。产品形态从"聊天界面"变成"任务看板"，UX 设计完全不一样。

**对技术架构师**。如果你现在在 Opus 4.6 的 max 档位上跑生产流量，立即测一下 Opus 4.7 的 xhigh 档位。如果效果持平，直接切——成本减半。不要等新模型稳定三个月才迁移，那三个月的 token 钱够给团队多雇一个人。

**对 Claude Code 重度用户**。`/ultrareview` 值得加入你的 PR workflow。不是每个 PR 都跑，而是关键模块（认证、支付、数据迁移）的 PR 强制跑一次。三次免费额度意味着 Anthropic 在用免费试用换你的使用习惯——用就是。另外 Auto Mode 是 Max 专享，如果你在 Pro 档，这次升级 Max 的 ROI 比以往都高。

**对 AI 安全研究者**。关注 Cyber Verification Program 的准入机制。它是 Anthropic 第一次正式对外建立"能力解锁许可证"体系。未来这种"专业豁免通道"会成为高风险 AI 能力的行业标配——值得理解它的申请逻辑和审核标准。

**对行业观察者**。Model Tiering 是未来两年的核心看点。观察 OpenAI 和 Google 会不会跟进类似分层，以及监管者（特别是 AISI、EU AI Office）会不会把这种分层写进合规模板。如果写进去了，那所有 foundation model 厂商都得这么做。

## 八、本期关键词

**Project Glasswing（玻璃翼项目）** — Anthropic 为 Mythos Preview 建立的受限分发联盟，目前包含 AWS、Apple、Google、Microsoft、NVIDIA、JPMorganChase、Cisco、CrowdStrike、Broadcom、Palo Alto Networks、Linux Foundation 共 11 家机构。为什么值得知道：这是第一个"最强 AI 模型只给联盟不给大众"的官方结构，未来会成为 AI 治理的参考案例。

**Differential Training（差异化训练）** — 在训练阶段有针对性地削弱某些能力，同时保留其他能力的技术。Anthropic 用它把 Opus 4.7 的攻击性网络能力降低到 Mythos 以下。为什么值得知道：这是"模型定制化"从 prompt 层下沉到训练层的关键技术，未来会催生行业专版模型（医疗版、金融版、教育版）。

**Task Budget（任务预算）** — 给 Claude 一个 token 预算（覆盖思考/工具调用/输出），模型看到 running countdown 并据此安排工作优先级。为什么值得知道：这是 agentic AI 从"不可预测花费"走向"可预算化生产"的关键产品设计。

**xhigh** — Claude 的推理 effort 档位，介于 high 和 max 之间。Opus 4.7 的 xhigh 在 100k token 时性能等同 Opus 4.6 max 在 200k token。为什么值得知道：它代表 price-performance 竞争进入"同等质量更省 token"的新阶段。

**/ultrareview** — Claude Code 中一个结构化代码审查命令，从架构、安全、性能、可维护性四个维度分析代码改动。Pro/Max 用户每月 3 次免费。为什么值得知道：这是 Anthropic 首次把特定工作流打包成独立命令产品化，是"后模型时代"产品分化的信号。

**Auto Mode（自主模式）** — Claude Code 的权限模式，允许 Claude 在权限范围内自己决策、不打断用户。仅限 Max 用户。为什么值得知道：它把 agentic loop 从"每轮需要用户批准"推进到"几小时无人值守"，是 AI 同事化的关键一步。

**Model Tiering（模型分级发布）** — 把一个底座模型的不同能力版本分层发布：Tier 1 给联盟（Mythos），Tier 2 公开商用（Opus 4.7），Tier 3 专业豁免（Cyber Verification Program）。为什么值得知道：它可能成为未来两年所有前沿 AI 厂商的默认发布结构。

**Cyber Verification Program（网络安全认证计划）** — Anthropic 为正当网络安全用途（漏洞研究、渗透测试、red-teaming）专业人员设立的访问申请通道，通过后可在 Opus 4.7 上解锁默认被关闭的部分能力。为什么值得知道：这是 AI 能力解锁许可证的第一个官方实现，是 "dual-use AI" 治理的实操样板。
