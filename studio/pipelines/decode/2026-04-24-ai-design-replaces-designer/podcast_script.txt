大家好，今天想聊一个刚发生不久的事。四月十七号到二十一号，五天之内，AI 圈发生了三件事，合起来讲了同一个故事——设计这件事，正在被悄悄地切成三段。

十七号，Anthropic 发布了 Claude Design。二十一号，OpenAI 把图像生成升级成会思考的 gpt-image-2。中间那天，Anthropic 的 CPO 辞掉了 Figma 的董事席位。这个时间点太精确了，不是巧合。

[转场]

先说 Claude Design。它做的事很有意思——它产出的不是图片，是代码。你给它一个 prompt，它吐出来的是可以直接跑的 HTML、CSS、React 组件，带交互、带动效。更关键的是，它能读你团队现有的 codebase，自动总结出一套设计系统：颜色、字体、组件库，之后所有项目都自动沿用。设计稿做好了，还可以打包成一个交接包——组件代码、设计 tokens、文案、交互说明——一键扔给 Claude Code 直接变成产品。

再说 gpt-image-2。它做的事看起来完全不同，还是在画图。但画图方式变了。旧模型拿到 prompt 就直接渲染，新模型会先读——读 prompt、规划构图、推理空间关系、必要时上网查实时信息、多图批处理时保持人物一致、最后还自己校验一遍输出。OpenAI 给它起了个名字叫"视觉思考伙伴"，把这套能力压进一个付费层级，叫 Thinking Mode。

两家做的事，一个在出代码、一个在出像素，表面看各走各的。但如果站远一步——Claude Design 把"把意图变成代码"这段活自动化了，gpt-image-2 把"把意图变成视觉"这段活智能化了。它们切入的都是同一个地方：设计师的中段工作。不是前面的"这事该不该做"，也不是后面的"品牌一致不一致"，而是中间那段"把想好的东西做出来"。

[转场]

这里有一个挺清楚的框架，设计这件事可以切成三段。判断段——这事值得做吗、给谁用、解决什么问题。执行段——把想法变成具体的设计。收尾段——把做出来的东西拧成一个品牌下的一致产物。AI 这一刀，砍得很准，全部落在执行段。

那市场怎么看这件事？这才是最有意思的地方。Claude Design 发布当天，Figma 股价跌了百分之七。年初到现在累计跌了百分之三十五。说 Figma 是 AI 工具里最惨的受害者不过分——它几乎是第一个被直接攻击工作流的独立设计工具公司。

比股价更说明问题的是人事动作。发布前一天，四月十六号，Anthropic 的 CPO Mike Krieger 辞去了 Figma 的董事席位。时间点太精确了——一个能在两家公司之间看到全部牌的人，在动手前一天先把自己从其中一家的董事会里抽出来。这是利益冲突管理。资本市场读到的信号很清楚：这次不是小打小闹。

更有意思的是 Canva 的选择。Canva 的 CEO 在 Claude Design 的发布稿里亲自站台，说"非常高兴跟 Claude 合作，让人们可以顺畅地把 Claude Design 里的草稿带进 Canva 编辑"。Canva 选择成为 Claude Design 的输出终点，而不是对手。这是典型的生态选择：在还来得及的时候选边站。Figma 当天对 Gizmodo 的问询没回应。一个拥抱，一个沉默，两种姿态都在告诉行业这事不小。

那 gpt-image-2 这边呢？reasoning 被嵌入图像生成流程，这个变化的分量容易被低估。

以前的图像模型是渲染器，理解 prompt，生成像素，一对一。prompt 写得不精确、排版逻辑复杂、要求文字清晰，输出就崩。新模型把这个链条改成了：读 prompt、规划 layout、推理空间关系、渲染像素、自己校验一遍。gpt-image-2 声称多语言文字准确率百分之九十九，一次最多可以出十张图，其中最多八张能保持人物一致，单图最高两千像素宽。把这些数字合起来看——它已经能独立完成一个"画完再修一遍"的循环，不需要人在旁边反复调 prompt。

这里值得命个名——意图到像素的短链路。以前这条路要经过"设计师翻译 prompt、工具渲染、人肉检查、重写 prompt"，要来回四到五次。现在模型内部自己完成一次完整循环。外面的人只需要给一个意图，等结果。

这条短链路，对谁最受益？答案不是设计师，是手上没设计师的产品人、PM、创业者——他们以前是"做不出来所以干脆不做"的那群人，现在这条路通了。

[转场]

有意思的是，一位真正的设计师给了挺冷静的判断。一位叫 Fanny 的设计师在 Medium 上发了一篇文章，标题叫"Claude Design 到底改变了设计师的什么"。她的判断比 Anthropic 的 PR 稿要冷静得多。

她承认工作流变了，而且是实质的变化——以前一个 idea 到一个功能上线，要走 brief、mockup、评审、对齐、交付，一整套冗长的流程；Claude Design 把这段压成一次对话。她引用了 Datadog 的反馈："一个星期的来回拉锯压缩进一次对话"。Brilliant 的数字是："旧工具要二十几个 prompt，Claude Design 只要两个。"这些数字摆出来很漂亮。但她紧接着抛了一句刹车——原话是"流水线优化吞吐量，但设计不总是一个吞吐量问题"。

这句话是全文的核心判断。她说，被压缩掉的那些"两像素反复调整、跟自己争论"的时间，不是效率损失，是创意摩擦——它本身在做事，只是肉眼看不见在做什么事。

她还对 Claude Code 做了一个观察：没有结构化规划时，Claude Code 一次性成功的概率大概只有百分之三十三；给它足够空间先写规范、讨论方案、推演边界，成功率会大幅提升。她的结论是：Claude Code 最大的强项不是速度，是规划深度。这个观察反过来证明了前面那条判断——判断段不能被压缩，能被压缩的只是执行段。

她列了一个"不会被替代"的清单：品味、判断、原则、对人类心理的理解、对商业战略的理解。这份清单的有意思之处在于，它恰好全部落在判断段和收尾段，没有一项落在执行段。换句话说，她完全同意"中段会被吃掉"，只是在提醒："别把前后两段也一起交出去。"

官方措辞里还有两个细节值得放大。

第一个——Anthropic 官方的定位原话是："Claude Design 是补充 Canva，不是替代它。"只字不提 Figma。但市场不会说假话——百分之七的跌幅、百分之三十五的年跌幅、CPO 辞职的时机，所有信号都指向同一件事。官方措辞是外交辞令，只看措辞会漏掉事实。

第二个——gpt-image-2 的分层设计。Instant Mode 免费，Thinking Mode 付费。这意味着"会思考"这个能力被明码标价了。图像生成从过去的"免费加限量"分层，变成现在的"快还是深思"分层。付费买的不是更多次数，是更深的推理。这个定价结构说明 OpenAI 认为 reasoning 本身是可以单独出售的商品。这在图像生成领域是第一次。

还有一个无法忽视的事实——两款产品目前都处于 Research Preview 状态。产品还没准备好到可以大规模稳定使用，边界场景会翻车，产出一致性还没保证。这条真话官方不会明说，但 Research Preview 这个标签本身就是在说。

[转场]

那对不同身份的人来说，这些变化意味着什么？

对产品经理来说，这是第一次，你可以不跪求设计师就能完成 pitch 稿、产品提案图、早期原型 demo。Claude Design 做可交互原型，gpt-image-2 做视觉素材。内部评审、融资 pitch、跨部门沟通这些场景，产出质量已经够用。但要注意：Research Preview 状态下，品牌细节、交互边界都不稳定，做 demo 可以，做 production 还得找人。这是一个"demo 可以自理，产品需要配合"的状态。

对设计师来说，中段被压缩是事实，这不是行业焦虑话术，是已经发生的产品现实。但这不是终点——护城河从"做得出来"转移到了"判断该做什么"和"把 AI 产出拧成品牌一致的东西"。前段需要产品直觉、用户理解、商业感；后段需要视觉判断、品牌一致性、细节把控。这两段都不是可以被下一代模型继续压缩的东西。学 prompt engineering、跑一遍 Claude Design 到 Claude Code 的 handoff 流水线——这些是把 AI 装进自己工具箱的操作，不是自贬，是升级。

对小团队和创业者来说，融资前、MVP、demo 阶段，不雇设计师是可行的路径。省下的是前期现金，但要意识到这是时间窗口，不是永久状态。产品一旦需要规模化、品牌资产、跨平台一致性，设计师还是必要的。把 AI 当作时间窗口的工具，而不是永久的解决方案。

[转场]

最后来收一下。设计师这个职业不会消失。设计师做的事情会被切成三段——判断段、执行段、收尾段。AI 能吃掉的是执行段，不是整个职业。Figma 这类中间态工具会被压缩，Canva 这类生活化工具选择融入生态，Claude 和 OpenAI 这类模型公司直接把 endpoint 打到创意工作流的入口和出口。工具层在重新分区，职业内部的价值重心也在迁移。

Fanny 在她的文章结尾写了一句话——"你仍然需要品味、判断、原则，以及理解人类心理和商业战略的人。"这句话看起来像是对设计师的安慰，但它其实是在描述设计这个职业 2026 年之后的新分工。

五天里发生的这些事，不是设计师的墓志铭，是设计这件事第一次被正式拆开看——拆开了才知道哪一段可以交给机器，哪一段还得自己来。知道这个，比焦虑谁会被替代要有用得多。

最后留三个关键词。第一个：设计三段分层——判断段、执行段、收尾段，AI 当前只切走执行段。第二个：意图到像素的短链路——gpt-image-2 把生成循环压进了模型内部。第三个：创意摩擦——反复调两像素的那段时间，是价值不是浪费。

好，今天就到这里。拜拜。
