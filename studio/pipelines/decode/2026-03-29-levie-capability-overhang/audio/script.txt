今天聊一个让我特别有感触的概念，叫 capability overhang，能力悬置。

这个词是 Box 的 CEO Aaron Levie 前两天在推特上提出来的。他写了一篇长文，核心就一句话：AI 的能力已经远远超过企业实际用到的水位了。浪已经打到岸边了，但企业的门还关着。

[转场]

你看编程领域，已经发生了不可逆的事。越来越多的团队公开说，我们的产品现在完全由 Agent 写成。模型写代码的能力进化速度，已经把传统开发工作流甩在身后了。

但你要是走出硅谷，去跟金融、法律、制造业的知识工作者聊一聊，完全不同的景象。AI 在他们那里，还就是个答题机器。问问题、查信息，仅此而已。大规模自动化？遥远得像科幻小说。

Wharton 的教授 Ethan Mollick 说了一句特别狠的话：就算今天停止一切 AI 研发，仅凭现有模型，也足以改变大部分白领工作。注意啊，他说的是现有模型，不是未来的。

所以问题不是 AI 不够好，是企业还没准备好接收它。这就是能力悬置。

[转场]

那为什么编程领域落地最快呢？Levie 引用了 Dwarkesh Patel 和 Dario Amodei 对话里的一个洞察，我觉得特别精准。

编程工作所需的全部上下文，就住在代码库里。函数定义、调用关系、测试用例、注释，全是机器可读的、结构化的。Agent 不需要去问任何人，自己就能找到工作所需的所有信息。

但其他知识工作不是这样的。

一个法律顾问的关键上下文，散落在几年前某次客户电话的记忆里、上司没有写下来的判断偏好里、竞争对手的动向里。这些东西不在任何数据库里，AI 根本看不到。

有个投资人说了一句话，我觉得是这篇文章里最精辟的一句：上下文一直是增长的瓶颈，因为任何拥有完整上下文的事情都可以被自动化。现在的竞赛，是让组织的集体大脑对 AI 变得可读。

说白了，这不是技术问题，这是信息基础设施问题。

[转场]

Levie 接着描述了三堵墙，我觉得每一堵都说到点子上了。

第一堵墙，数据在错误的地方。大量企业数据还躺在本地服务器、老旧系统里，跟云端的 Agent 物理上没法对话。云化迁移喊了十几年了，到 2026 年还没搞完。

第二堵墙，权限比你想象中复杂一万倍。在真实的组织里，没有两个人拥有完全相同的信息访问权限。这是设计如此的。但当 Agent 开始以一百倍、一千倍于人类的速度操作系统的时候，权限管理的复杂度就爆炸了。有时候 Agent 需要更宽的权限，有时候需要更窄的。两种情况目前都没有成熟解法。

第三堵墙，技术标准在跑步，企业在爬行。十八个月前最佳实践是向量数据库加 RAG，然后 MCP 出来了，然后 CLI 原生 Agent 出来了。企业以年为单位做 IT 决策，AI 工具以月为单位迭代。永远在追赶。

我把这三堵墙合在一起，起了个名字，叫上下文债。就像技术债是多年不规范编码积累的代价，上下文债是多年非结构化、碎片化信息积累的代价。企业要部署 AI Agent，必须先还这笔债。

[转场]

那谁会赢呢？Levie 的判断我觉得很清晰：在 AI 能力不再是瓶颈的世界里，真正的竞争优势在于接线能力。谁能把 AI 的智力接入真实的企业工作流，谁就赢。

他给了三类赢家。

第一类，已有平台，比如他自己的 Box。已经深嵌在客户工作流里了，已经有数据信任了，只需要把 AI 接进现有管道。这是最快的路径。

第二类，新兴垂直 Agent。从零重新设计特定行业工作流的新玩家，不受遗留系统束缚。

第三类，这个有意思了，系统集成商和咨询公司。Levie 特别说了一句：Don't write off the SIs yet。别急着写掉 SI。大企业要跑通 AI Agent，需要有人来做深度的变革管理，这是重度专业服务，产品替代不了。

[转场]

最后 Levie 做了一个让人清醒的类比。

2010 年，硅谷所有人都知道云计算是未来。那年 AWS 收入大概五亿美元。到 2025 年，三大云平台合计两千两百五十亿美元。十五年，增长接近一千倍。而且还在高速增长。

从所有人都知道这是未来，到市场真正成熟，花了十五年。

Agent 不会花十五年，但也不会花十五个月。扩散慢，反而意味着机会窗口长。现在入场的玩家，有足够时间建立竞争壁垒。

[转场]

最后说说这篇文章的盲区。Levie 整体偏乐观，有两个地方我觉得值得质疑。

第一，上下文工程的成本被低估了。把企业的非结构化数据迁移到 AI 可读的格式，不只是技术问题，还涉及组织变革、数据治理、人员培训。大型企业真实的迁移周期往往三到五年。

第二，权限问题在监管行业可能是硬墙。金融、医疗、法律，权限和合规要求不是技术难题，是法律红线。有些企业可能永远没法给 Agent 足够的访问权限。

但总的来说，capability overhang 这个框架非常有价值。它告诉我们：最稀缺的不是 AI 能力，而是懂 AI 又懂企业运作的人。能在两者之间做翻译的人，不管在哪个组织里，都会变得极其有价值。

好，今天就聊到这儿。三个关键词带走：能力悬置、上下文债、接线员。下次见。
