# Phase 1: 结构解剖 — claude-code-leak-panorama

## 1. 核心论点（一句话）

Claude Code 的源码泄露揭示了一个事实：AI 编程助手的竞争力不在模型（harness > model），而在于围绕模型构建的工程体系——六层自研架构、六级安全、双 AI 制衡、记忆系统——这套体系的复杂度和精密程度远超行业想象。

## 2. 隐含假设

- **"源码泄露 = 护城河被削弱"** 是浅层判断。深层判断是：工程判断力和组织文化不可复制。
- **Anthropic 从第一天就把安全当地基** — 暗示其他竞品没有做到这一点。
- **"聊天机器人加壳"的产品和 Claude Code 之间是认知差距** — 将 AI 编程助手市场分为两个档次。

## 3. 关键证据

1. **59.8MB source map → 1902 TS 文件 / 51 万行代码** — 技术事实，确凿
2. **六层架构，全栈自研** — QueryEngine 13K 行，Ink UI 80+ 文件，核心链路零外部依赖
3. **双 AI 安全架构** — 主 AI 执行 + 独立 Sonnet(T=0) 分类器 + 四道流水线 + 熔断机制
4. **四个隐藏功能** — Buddy/KAIROS/Undercover/Anti-Distillation，构成产品路线图
5. **同款错误犯两次** — 2025-02 v0.2.8 和 2026-03-31 v2.1.88
6. **3000 行函数** — print.ts 的屎山证据
7. **正则表达式情绪检测** — 不用 AI 做情感分析

## 4. 盲区（原文没说或轻描淡写的）

- **开源复刻版的实际能力** — 文章提到多个 fork 但没评估它们的质量
- **对 Anthropic 商业影响的量化分析** — "13-20% 年收入"是媒体估算，没有独立验证
- **Buddy 系统的工程成本** — 用混淆躲过内部扫描，说明 Anthropic 内部流程有问题
- **为什么第二次还犯** — 深层根因分析不够（是 CI/CD 流程问题？是代码审查问题？）
- **合规/法律影响** — DMCA 下架的实际效力，GPL/开源社区的反应

## 5. 读者画像

- AI 从业者：想了解顶级 AI 产品的内部架构
- 竞品工程师：想抄作业
- 投资/战略分析：想评估 Anthropic 的护城河
- 对我们（AIForce 用户/实践者）意味着什么：理解 harness > model，把精力放在工程体系上
