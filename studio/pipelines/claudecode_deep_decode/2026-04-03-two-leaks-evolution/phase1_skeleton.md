# Phase 1: 结构解剖

## 1. 核心论点（一句话）
同一个配置错误，14个月，两次泄露——但两次暴露的"遗产"规模差了两个数量级，这个差距本身就是 AI 编程工具进化速度的最佳证据。

## 2. 隐含假设
- 假设读者知道 npm 是什么、sourcemap 的基本概念
- 假设读者对 Claude Code 有基本了解（当红 AI 编程工具）
- 原文（多信源）隐含假设：泄露是坏事。但反面论证：泄露实际上让社区深入理解了 Agent 架构的最佳实践
- 12版本逆向文章隐含假设：手工逆向比 sourcemap 提取更有技术深度（值得验证）

## 3. 关键证据
- v0.2.8 (2025-02): 单文件 cli.mjs，base64 sourcemap，ghuntley 用 LLM cleanroom 反混淆
- v2.1.88 (2026-03-31): 1902 文件，512K+ 行，59.8MB cli.js.map，.npmignore 缺失
- 12版本逆向 (kolkov): v2.1.74-v2.1.88，手工解析，ccdiag 工具分析 148,444 次工具调用
- Bun bundler bug: development:false 仍生成 sourcemap
- 50,000 stars in 2 hours = GitHub 历史最快增长仓库
- 同日 axios 供应链攻击，时间窗口重叠

## 4. 盲区
- ghuntley 原文 paywalled，早期版本的具体技术细节有限
- v0.2.8 到 v2.1.88 之间的中间版本演进缺少公开分析（kolkov 只覆盖了最后12个版本）
- 没有 Anthropic 官方的事故复盘（只有"发布打包人为错误"一句话）
- 第一次泄露（2025-02）的社区反应远小于第二次，缺少对比数据

## 5. 读者画像
- AI 从业者/产品人：关心技术演进速度、架构决策、安全启示
- 开发者：关心 npm 发布安全、sourcemap 机制、逆向方法论
- 围观群众：关心八卦（同一错误犯两次、Undercover Mode 反讽）
