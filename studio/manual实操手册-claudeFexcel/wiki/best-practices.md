# 最佳实践 — Claude for Excel / PowerPoint

## 使用原则

四条铁律，贴在工位上：

1. **始终审查变更再定稿** — Claude 的修改可能不完全符合预期。保存前通读一遍，尤其是公式和数字
2. **验证输出匹配组织方法论** — 不同公司的财务计算标准可能不同（比如折旧方法、收入确认规则），Claude 不知道你公司的规矩
3. **使用适当的权限和访问控制** — 尤其在 Team / Enterprise 环境，别让不该看到数据的人看到
4. **对客户交付物保持人工监督** — Claude 是你的助手，不是你的替身。发给客户的东西你要负责

## Excel 专项建议

- **复杂操作前先保存**（Ctrl+S）— Claude 改错了你还能恢复
- 用 **"Claude Log" 标签页**追踪 Claude 的操作记录 — 这是 Claude 自动创建的日志页，记录了它做了什么
- 出错随时用 Excel 标准撤销（**Ctrl+Z**）— 和平时撤销操作一样
- **告诉 Claude 具体要修改的区域**，而不是让它自己找。比如说"修改 B3:B12 的公式"，不要说"帮我看看哪里有问题"

## PowerPoint 专项建议

- **先应用模板，再让 Claude 生成内容** — 这样 Claude 会沿用你的模板风格，而不是用默认样式
- **具体说明要修改哪张幻灯片或哪个元素** — 比如"修改第 3 页的标题"而不是"改一下标题"
- **验证输出符合品牌指南** — Claude 不知道你公司的 logo 用什么颜色、标题用什么字体

## Prompt 技巧

写给 Claude 的指令越清楚，结果越好：

### 说清楚目标和约束

```
好的例子：把 B3:B12 的数值格式改为千分位，保留 2 位小数，保持现有公式不变
不好的例子：帮我格式化一下表格
```

### 用具体的单元格引用

```
好的例子：在 D2 单元格写一个 VLOOKUP 公式，从 Sheet2 的 A:C 区域查找 A2 的值
不好的例子：帮我做一个查找公式
```

### 分步骤给复杂指令

不要一次性塞太多要求。把大任务拆成小步骤：

1. 先让 Claude 理解数据结构
2. 再让它执行第一步操作
3. 确认结果后继续下一步

### 利用持久指令（Instructions）

在 Claude 侧边栏的 Instructions（指令）区域，可以设置**反复用到的偏好**。比如：

- "所有金额显示为人民币格式"
- "公式中使用中文函数名"
- "输出结果保留到小数点后两位"

这些指令会在整个对话过程中持续生效，不用每次重复说。

## 安全守则

- **只用可信来源的电子表格** — 不要用网上下载的、供应商发来的、来路不明的文件（详见 [[限制与安全]] 的提示注入章节）
- **审查 Claude 对外部数据的获取请求** — 看到确认弹窗时仔细看清楚要访问什么
- **不要在不可信文件中使用 Claude** — 恶意单元格内容可能诱骗 Claude 执行危险操作
- **连接器只连可信的 MCP 服务器** — MCP（Model Context Protocol，模型上下文协议）连接器能让 Claude 访问外部数据源，确保这些数据源是你信任的

## 用量优化

Claude 的使用额度是有限的，以下方法能让你用得更久：

| 策略 | 效果 |
|------|------|
| 用 Projects + RAG（检索增强生成）处理大数据集 | 避免把所有数据塞进单次对话 |
| 不需要时关闭扩展思考（Extended Thinking）、网络搜索 | 这些功能会消耗额外 token |
| 保持项目指令简洁 | 指令越长，每次对话消耗的 token 越多 |
| 定期清理不再使用的项目文件 | 减少不必要的上下文加载 |
