# Excel 核心功能

Claude for Excel 的所有核心能力详解。每个功能包含：做什么、怎么用、注意什么。

---

## 1. 读取和理解模型

### 功能描述

对着打开的工作簿直接提问，Claude 能读取单元格内容、公式结构、多标签页（Sheet）之间的关系，并在回答中引用具体的单元格位置。你不需要告诉它数据在哪，它会自己找。

适用场景：接手别人的模型、快速理解复杂工作簿的逻辑、找到关键假设藏在哪个 Sheet 的哪个角落。

### 示例 Prompt

| 你想做的事 | 对 Claude 说 |
|------------|-------------|
| 找到关键假设 | "Q3 收入预测的驱动假设是什么？" |
| 理解公式逻辑 | "解释 WACC 计算如何流入 DCF 模型" |
| 跨 Sheet 查找 | "哪些标签页引用了 Revenue 这个 Sheet？" |

### 使用提示

- Claude 会在回答中标注单元格引用（如 B15、Sheet2!D3），点击可跳转
- 支持 .xlsx 和 .xlsm 格式
- 多标签页工作簿也能处理，Claude 会自动导航各 Sheet

---

## 2. 安全更新假设

### 功能描述

修改财务模型中的假设参数（比如增长率、利率、折现率），Claude 会维护公式依赖关系（Formula Dependencies，公式之间的引用链条）。改一个数不会把下游一串公式搞坏。

每次修改后，Claude 会：
- 高亮所有被更新的单元格
- 解释每个变更的影响
- 保留原有的公式结构和格式

### 示例 Prompt

| 你想做的事 | 对 Claude 说 |
|------------|-------------|
| 调整增长假设 | "把增长率提高 2%，展示对终值的影响" |
| 更新利率 | "根据最新美联储指引更新利率假设" |
| 场景分析 | "创建基础、乐观、悲观三种场景，用不同增长假设" |

### 使用提示

- 修改前 Claude 会警告是否会覆盖已有数据（Overwrite Protection）
- 所有更改都可以用 Excel 标准撤销功能（Ctrl+Z / Cmd+Z）回退
- 建议一次只改一组相关假设，方便追踪影响

---

## 3. 构建和填充模板

### 功能描述

两种用法：

1. **从零建模** — 用一句话描述需求，Claude 从空白表格搭出完整的财务模型、数据分析表，包含公式、格式和标签页结构
2. **填充现有模板** — 上传数据源（PDF、其他 Excel），Claude 把数据映射到模板的对应位置

### 示例 Prompt

| 你想做的事 | 对 Claude 说 |
|------------|-------------|
| 从零建模 | "为 SaaS 公司构建三表模型（利润表、资产负债表、现金流量表）" |
| 填充模板 | "用上传的 10-K 数据填充这个 DCF 模板" |
| 定制模型 | "建一个 LBO 模型，包含债务计划和回报分析" |

### 使用提示

- Claude 能识别常见财务建模模式和行业标准计算方式
- 从零建的模型建议先跑一遍检查公式逻辑，再用于正式工作
- 填充模板时，Claude 会自动匹配数据字段和模板位置

## 4. 调试和修复错误

### 功能描述

遇到 Excel 报错时，Claude 能定位错误根源并提供可操作的修复建议：

| 错误类型 | 含义 | Claude 怎么帮 |
|----------|------|---------------|
| `#REF!` | 引用错误（被引用的单元格被删了） | 定位断裂的引用，建议修复路径 |
| `#VALUE!` | 值类型错误（比如文本和数字混算） | 找出类型不匹配的位置 |
| 循环引用（Circular Reference） | A 引用 B、B 又引用 A 的死循环 | 画出引用链条，指出断环点 |

### 示例 Prompt

| 你想做的事 | 对 Claude 说 |
|------------|-------------|
| 诊断单个错误 | "为什么这个 NPV 计算返回 #VALUE?" |
| 全局扫描 | "找出这个工作簿中所有循环引用" |
| 公式排查 | "B4 单元格报错了，帮我追踪问题" |

### 使用提示

- Claude 修复错误时会保持电子表格的完整性，不会乱改周围的公式
- 对于复杂的循环引用，Claude 会解释引用链条，帮你理解为什么会循环
- 修复后建议检查一下相关联的单元格是否也恢复正常

---

## 5. 变更追踪和引用

### 功能描述

Claude 在操作你的工作簿时，会自动做两件事：

1. **高亮变更** — 每个被更新的单元格都会高亮标记，附带解释性注释（为什么改、改了什么）
2. **可点击引用** — 当 Claude 解释计算逻辑时，提到的单元格引用可以直接点击跳转到对应位置

这意味着你不需要自己去比对"Claude 到底改了哪些格子"，它会主动告诉你。

### 使用提示

- 可以手动请求 Claude 创建"Claude Log"标签页，记录所有操作日志
- 所有变更都支持 Excel 标准撤销
- 建议在保存文件前，先过一遍 Claude 高亮的变更区域

## 6. 原生编辑和格式化

### 功能描述

Claude 支持直接执行 Excel 原生操作，不是生成宏或 VBA 代码，而是直接帮你操作：

| 操作类型 | 说明 |
|----------|------|
| 排序和筛选（Sort/Filter） | 按条件排列、筛选数据 |
| 数据透视表（Pivot Table） | 修改现有透视表的字段、布局 |
| 图表（Chart） | 调整图表类型、数据范围、样式 |
| 条件格式（Conditional Formatting） | 根据条件自动变色、加图标 |
| 数据验证（Data Validation） | 设置下拉菜单、输入限制 |
| 打印设置（Print Setup） | 设置打印区域、网格线、页面布局 |

### 示例 Prompt

| 你想做的事 | 对 Claude 说 |
|------------|-------------|
| 排序 | "按收入降序排列这个表格" |
| 条件格式 | "给低于目标阈值的单元格加红色条件格式" |
| 下拉菜单 | "为状态列设置下拉菜单：Active、Pending、Closed" |
| 打印设置 | "关闭网格线，设置打印区域为 A1:F20" |
| 图表 | "创建一个瀑布图展示收入桥接" |
| 专业样式 | "用投行惯例格式化（蓝色输入、黑色公式）" |

### 使用提示

- 这些操作是原生 Excel 操作，结果和你手动操作一样
- 不支持宏（Macro）和 VBA 操作
- 格式化操作同样会高亮变更，可撤销

---

## 7. 文档导入

### 功能描述

Claude 可以从外部文档提取数据并导入 Excel：

1. **PDF 提取** — 从 PDF 中识别表格结构，转换为 Excel 格式
2. **模板填充** — 从上传的文档中提取数据，自动填入你的模板

### 示例 Prompt

| 你想做的事 | 对 Claude 说 |
|------------|-------------|
| PDF 转 Excel | "从这个 PDF 中提取财务表格到 Excel" |
| 发票提取 | "把这张发票 PDF 的行项目拉到我的模板里" |
| 模板填充 | "用这份投资备忘录的数据填充我的交易模板" |

### 使用提示

- PDF 提取依赖文档质量，扫描件的识别率可能低于原生 PDF
- 提取后建议检查数字精度，特别是小数点和千分位
- 支持将提取的数据映射到已有模板的标准格式
