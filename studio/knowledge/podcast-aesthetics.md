# 播客听感美学

> **性质**：知识文件（LLM Wiki）。定义"好播客长什么样"。
> 执行脚本 / ffmpeg 命令 / TTS API 在 `.claude/skills/deep-decode/references/podcast-pipeline.md`。

---

## 标杆

**声动早咖啡**（小宇宙 FM）—— 所有设计决策以它为参照。

---

## 听感目标

节奏快不拖拉 / BGM 不抢戏 / 像朋友在聊天 / 轻松的氛围传递好玩有用的严肃知识。

---

## 设计决策表

| 设计决策 | 做法 | 为什么 |
|---------|------|--------|
| 开场 | 固定品牌片头（音乐+人声，3-5 秒） | 建立品牌识别 |
| 进入正文 | 开头 2-3 句话说重点，直接进精解 | 不废话，尊重时间 |
| 转场 | 纸张翻动音效（~0.5 秒） | 轻提示换话题，不打断 |
| BGM | 全程铺底，正文部分音量极低（降 25-30dB） | 有氛围感但不抢戏 |
| 语速 | 快但清晰，像朋友聊天不像念稿 | 密度感 + 亲切感 |
| 结尾 | 说完最后要点 → 短结语 → 品牌片尾（2-3 秒） | 干脆利落 |
| 时长 | 10-12 分钟 | 一段通勤刚好 |
| 脚本字数 | 2500-3000 字（中文朗读 ~300 字/分钟） | 严格控制 |

---

## 播客结构模板

```
[0:00-0:04]  品牌片头（固定音乐+人声："AI 前沿拆解"）
[0:04-0:15]  2-3 句话说最重要的结论（钩子）
[0:15-正文]  逐章节精解
             章节间：翻页音效（0.5s）+ 200ms 停顿
[结尾前 30s] "最后总结三个关键词：第一... 第二... 第三..."
[结尾]       短结语 → 品牌片尾（2-3 秒）
```

全程 BGM 铺底，正文部分极低音量（-28dB），不抢人声。

---

## 响度标准

- 播客标准响度：-16 LUFS
- LRA 目标设为 4（参考声动早咖啡的 4.1），让声音更稳定
- 动态压缩让声音更"稳"，不忽大忽小

---

## 声音选型

### 单人模式

- **男声**（沉稳但不沉闷）：`zh_male_jingqiangkanye_moon_bigtts`
- **女声**（自然活泼）：`zh_female_roumeinvyou_emo_v2_mars_bigtts`

### 双人模式

- **A（男，理性分析派）** + **B（女，好奇追问派）**
- 对话自然有互动（"等等你刚才说的意思是..."、"对而且更有意思的是..."）
- 每段 2-4 句话，15-25 轮

---

## 品牌片头

**声音必须跟主讲人一致**——男声播客用男声片头，女声播客用女声片头。不能所有版本都用同一个片头。

生成播客时按 voice_type 同步生成对应的片头音频：`intro_male.mp3` / `intro_female.mp3`。

片头内容："AI 前沿拆解" + 3 秒轻快音乐。

---

## 脚本文字风格

- 口语化连接词："说到这儿"、"有意思的是"、"回过头看"、"其实"、"对吧"
- 语气词适量："嗯"、"那"、"所以呢"——让节奏更自然
- 不用"我认为"，用"数据表明"、"这意味着"、"值得注意的是"
- 开头 2-3 句话直接说最重要的结论，不铺垫
- 结尾总结 3 个关键词/要点
- 话题切换处标注 `[转场]`（TTS 时跳过，拼接时换成翻页音效）

---

## 基调（跟正文一致）

**不贩卖焦虑**——技术解析类内容的基调是"这很有意思，值得关注"，不是"天要塌了快跑"。

详见 `writing-style.md` §不贩卖焦虑。

## 短视频字幕美学（如果出视频版）

短视频平台（小红书/抖音/视频号）大量用户静音刷视频，**没字幕 = 没人看**。

- 必须硬字幕（burnt-in），不依赖平台自动识别
- 白字黑边，居中底部
- 竖版 FontSize=14，横版 FontSize=12，MarginV=20
- SRT 按句号/问号/感叹号断句，不在句中截断
- 逗号处可断但仅当单句超 30 字
- 每条字幕必须是完整语义单元
