# 概念可视化 SVG 设计规范

> **性质**：知识文件（LLM Wiki）。Skill 执行到画图阶段时读取本文件。

---

## 核心原则

**图形不是装饰，是独立可传播的知识单元。** 一张图发到群里，不看正文也能 get 核心概念。

图形排列要符合人的认知规律：总→分→总、因→果、旧→新。

---

## 双风格体系

### 风格 A：概念漫画图

用图形隐喻讲解抽象 AI 概念。不一定画小人，核心是**概念可视化**。

**适用**：对比（旧 vs 新）、隐喻（容器/管道/齿轮）、因果关系、概念演变

**视觉特征**：
- 手绘线条感：`stroke-linecap="round"` `stroke-linejoin="round"`，线条可略有不规则
- 可有简笔角色（圆头、简单表情），也可纯图形隐喻
- 底色可用极浅暖色 #FAFAF8 或白底
- 强调色：#37352F（深色主体）+ 1-2 个点缀色

**布局参考**：
- 左右对比（旧 vs 新，一眼看懂核心主张）
- 上→下因果链（原因→结果→影响）
- 中心辐射（核心概念→多个推论）
- 底部 1-2 句金句总结

### 风格 B：结构化图形（Notion 式日间主题）

流程图、矩阵、层级关系、数据对比。干净、现代、留白。

**适用**：流程图、架构图、数据对比、矩阵、时间线

**配色（Notion 日间主题）**：

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | #FFFFFF 或 #F7F7F5 | 画布底色 |
| 正文 | #37352F | 主要文字 |
| 次要文字 | #787774 | 标签、注释 |
| 强调蓝 | #2383E2 | 主要强调、链接 |
| 警告红 | #EB5757 | 风险、否定 |
| 成功绿 | #4DB6AC | 正面、完成 |
| 注意黄 | #FFB020 | 提醒、待定 |
| 特殊紫 | #9B51E0 | 创新、独特观点 |
| 边框 | #E3E2E0 | 卡片边框、分割线 |
| 卡片底色 | #F7F7F5 | 内容区块背景 |

**视觉特征**：
- 直线、圆角矩形（rx=8）、规整排列
- 充分留白，元素间距 >= 16px
- 无多余装饰（无阴影、无渐变、无 3D 效果）
- 箭头简洁：`marker-end` 三角形，stroke-width=2

## 共通规则

### 字体（关键，错了中文全是方框）

**必须**：`font-family="PingFang SC, Inter, sans-serif"`

**中文字体在前**。cairosvg 的 font-family fallback 机制要求中文在前，否则中文全渲染为方框。

- 正确：`font-family="PingFang SC, Inter, sans-serif"`
- 错误：`font-family="Inter, PingFang SC"` → 中文全是方框
- 错误：`font-family="Inter, 'PingFang SC'"` → 引号无效

### 尺寸

- 宽度固定 800
- 高度按内容适配：400-620
- 标题字号 20-24，正文 14-16，注释 12

### 信息密度

每张图必须包含：
- 核心概念（一句话，加粗或大字）
- 支撑内容（数据/案例/对比，至少 1 个）
- 逻辑关系（对比/流程/层级，至少 1 种）

### 认知规律排列

- **总分总**：顶部大标题 → 中间多个子块 → 底部总结
- **因果链**：左→右 或 上→下，箭头引导视线
- **对比**：左右并列，差异高亮
- **层级**：外→内 或 上→下，包含关系

### 编号

- `00_系列封面.svg` -- 每篇都有
- `01_xxx.svg` -- 按正文章节顺序，中文描述命名

## SVG 渲染约束（踩过的坑）

### 特殊 Unicode 符号渲染为方框

`✕` `✓` `√` `☑` 在 cairosvg + PingFang SC 下渲染为方框。用文字替代。`→` 是安全的。

### 中文字符宽度

中文字符宽度 ≈ 英文 x1.5。SVG 中 `<rect>` 内放中文文本时：
- 宽度按"中文字数 x 14px"估算（font-size=13 时）
- 用 `text-anchor="middle"` 居中比左对齐更不容易溢出
- 如果文字可能很长，用 font-size=11-12 而不是 13
- 容器 rect 宽度至少比文本预估宽度多 20px 余量
- 测试：渲染后检查 PNG，中文文本是否被容器截断

### marker 的 orient 属性

只能用 `orient="auto"`，不能用 `orient="auto-start-auto"`。后者在 cairosvg 会报 `could not convert string to float`。

### 不嵌套 foreignObject

cairosvg 不支持，用多个 `<tspan>` 手动换行。

### SVG 转 PNG 必须 2x

`cairosvg.svg2png(scale=2)` 确保高清。1x 在 Word / 微信 / 小红书中模糊。

## 设计自检（7 项）

1. 删掉文字后形状本身还能传递概念吗？
2. 字号有 3 级以上明显跳跃吗？（28 / 14 / 10）
3. 有没有超过 15% 面积的留白？
4. 能一眼找到 hero element 吗？
5. 颜色不超过 3 种语义色？
6. 2 秒扫描能 get 核心观点吗？
7. 发到群里不看正文别人能理解吗？
